# 영어 학습 앱 기획 초안

## 1. 제품 방향

이 앱은 사용자가 앱을 켜자마자 바로 영어 문제를 풀 수 있게 만드는 단순한 학습 앱이다.

복잡한 홈 화면, 긴 진단 과정, 여러 메뉴 선택을 먼저 보여주지 않는다. 앱은 로컬에 저장된 사용자의 현재 상태를 불러오고, 그 상태에 맞는 문제를 즉시 보여준다.

핵심 흐름은 다음과 같다.

```text
앱 실행
-> 로컬 학습 상태 로드
-> 현재 단계에 맞는 문제 표시
-> 문제 풀이
-> 해설 확인
-> 레이트 산정
-> 기준 도달 시 승급 시험
-> 승급 또는 유지
```

## 2. 핵심 목표

- 사용자가 앱을 열면 바로 문제를 풀 수 있다.
- 사용자의 현재 단계와 레이트는 로컬에 저장된다.
- 문제를 다 풀면 해설과 함께 레이트가 표시된다.
- 레이트가 일정 기준을 넘으면 바로 승급 시험으로 이동한다.
- 승급 시험은 일반 문제와 다른 UI로 보여준다.
- 승급 시험에 통과하면 다음 단계로 올라간다.
- 통과하지 못하면 현재 단계에서 계속 문제를 푼다.

## 3. 사용자 경험 원칙

### 3.1 첫 화면은 문제

앱의 첫 화면은 메뉴가 아니라 문제다.

사용자는 앱을 켜고 바로 다음 행동을 할 수 있어야 한다.

```text
앱 열기
-> 문제 보기
-> 답 선택 또는 입력
-> 제출
```

### 3.2 학습은 짧게

한 번의 일반 학습은 짧은 문제 묶음으로 구성한다.

권장 구성:

- 한 세션당 3문제
- 문제를 모두 풀면 결과 화면으로 이동
- 결과 화면에서 해설과 레이트 표시

### 3.3 승급 시험은 특별하게

승급 시험은 일반 문제와 구분되어야 한다.

일반 문제는 편안하고 가벼운 화면으로 보여준다. 승급 시험은 더 집중해야 하는 화면으로 보여준다.

승급 시험 UI 차이:

- 더 어두운 배경
- 승급 표시 배지
- 진행률
- 제한 시간 또는 남은 문제 수
- 해설 없이 시험 종료 후 결과 표시

## 4. 로컬 학습 상태

앱은 서버 계정 없이 로컬에 사용자의 현재 학습 상태를 저장한다.

저장할 데이터:

```ts
type LocalLearningState = {
  currentLevel: 'A1' | 'A2' | 'B1' | 'B2';
  currentRate: number;
  solvedQuestionCount: number;
  promotionReady: boolean;
  recentResults: RecentResult[];
  updatedAt: string;
};

type RecentResult = {
  questionSetId: string;
  level: 'A1' | 'A2' | 'B1' | 'B2';
  score: number;
  rateAfter: number;
  solvedAt: string;
};
```

로컬 상태가 없으면 기본 상태를 만든다.

```text
currentLevel: A1
currentRate: 0
promotionReady: false
```

## 5. 문제 출제 방식

앱은 현재 레벨과 레이트를 기준으로 문제를 고른다.

문제 선택 기준:

- 현재 레벨에 맞는 문제
- 최근에 틀린 유형과 비슷한 문제
- 너무 자주 나온 문제는 제외
- 승급 직전이면 조금 더 어려운 문제 포함

일반 학습 문제는 다음 형태를 우선 사용한다.

- 보기 선택형
- 빈칸 채우기
- 짧은 문장 수정
- 짧은 답변 입력

초기 MVP에서는 문제 수를 작게 시작한다. 문제 품질과 레이트 계산이 안정되면 문제 수를 늘린다.

## 6. 일반 문제 화면

일반 문제 화면은 최대한 단순해야 한다.

화면 구성:

- 상단: 현재 레벨과 레이트
- 중앙: 문제
- 하단: 답 선택 영역
- 맨 아래: 제출 버튼

예시:

```text
A1    Rate 42

가장 자연스러운 문장을 고르세요.

[ A. I want a coffee. ]
[ B. I want coffee a. ]
[ C. I wants a coffee. ]

[ 제출 ]
```

## 7. 해설과 레이트 화면

세션의 모든 문제를 풀면 해설과 레이트가 나온다.

화면 구성:

- 이번 세션 점수
- 새 레이트
- 문제별 해설
- 틀린 이유
- 다음 행동 버튼

예시:

```text
Rate 74

이번 문제 묶음에서 3문제 중 2문제를 맞혔습니다.

해설
- "I want a coffee."가 자연스럽습니다.
- "coffee a"는 관사의 위치가 잘못되었습니다.

[ 다음 문제 ]
```

레이트가 기준 이상이면 다음 버튼은 승급 시험으로 바뀐다.

```text
[ 승급 시험 시작 ]
```

## 8. 레이트 계산

레이트는 사용자의 현재 실력을 나타내는 숫자다.

기본 범위:

```text
0 ~ 100
```

일반 문제를 풀 때마다 레이트가 조금씩 변한다.

예시 규칙:

- 모든 문제 정답: +10
- 대부분 정답: +6
- 절반 정도 정답: +2
- 대부분 오답: -4

레이트는 현재 레벨 안에서만 계산한다. 다음 레벨로 올라가는 것은 레이트가 아니라 승급 시험 결과로 결정한다.

## 9. 승급 조건

레이트가 일정 기준에 도달하면 승급 시험으로 이동한다.

기본 기준:

```text
Rate 80 이상
```

조건을 만족하면 결과 화면에서 바로 승급 시험 버튼을 보여준다.

```text
Rate 82
승급 시험을 볼 수 있습니다.

[ 승급 시험 시작 ]
```

사용자가 승급 시험을 보지 않고 다음 일반 문제를 풀 수도 있게 할지는 다음 단계에서 결정한다.

## 10. 승급 시험

승급 시험은 현재 레벨을 넘어 다음 레벨로 갈 수 있는지 확인하는 시험이다.

승급 시험은 일반 문제와 다르게 동작한다.

- 해설은 시험 중 제공하지 않는다.
- 문제 수가 일반 세션보다 많다.
- UI가 더 진지하게 바뀐다.
- 시험 종료 후 결과만 보여준다.

권장 구성:

- 승급 시험 문제 수: 5문제
- 통과 기준: 80점 이상
- 통과 시 다음 레벨로 승급
- 실패 시 현재 레벨 유지

승급 시험 화면 예시:

```text
승급 시험
A1 -> A2

문제 2 / 5
남은 시간 03:20

가장 자연스러운 문장을 고르세요.

[ A ]
[ B ]
[ C ]

[ 시험 제출 ]
```

## 11. 승급 결과

승급 시험 결과는 단순하게 보여준다.

통과한 경우:

```text
승급 성공
A1 -> A2

새로운 단계의 문제가 시작됩니다.
```

실패한 경우:

```text
현재 단계 유지

조금 더 연습한 뒤 다시 승급 시험에 도전하세요.
```

실패해도 사용자를 강하게 벌주지 않는다. 현재 레벨에서 다시 문제를 풀면서 레이트를 회복하게 한다.

## 12. MVP 범위

포함할 기능:

- 앱 실행 시 로컬 상태 로드
- 로컬 상태가 없으면 기본 상태 생성
- 현재 레벨에 맞는 문제 표시
- 일반 문제 세션
- 문제별 해설
- 레이트 계산
- 승급 가능 상태 표시
- 승급 시험
- 승급 성공 또는 유지 처리
- 로컬 상태 저장

제외할 기능:

- 로그인
- 서버 동기화
- 복잡한 홈 화면
- 긴 초기 진단
- 음성 입력
- 발음 평가
- 랭킹
- 결제

## 13. 기술 방향

초기 구현은 단순한 로컬 중심 구조로 간다.

권장 구조:

- 로컬 저장: `AsyncStorage`
- 문제 데이터: 앱 내부 seed data
- 일반 문제 채점: 로컬 로직
- 짧은 답변 평가: 필요할 때만 서버 평가 사용
- API Key: 앱에 저장하지 않고 서버에서만 관리

초기에는 모든 문제를 AI가 만들 필요가 없다. 안정적인 seed 문제로 먼저 학습 흐름을 만든다.

## 14. 성공 기준

초기 MVP는 다음 조건을 만족하면 성공으로 본다.

- 앱을 켜면 바로 문제가 나온다.
- 문제를 다 풀면 해설이 나온다.
- 해설 화면에 레이트가 표시된다.
- 레이트가 기준 이상이면 승급 시험으로 이동할 수 있다.
- 승급 시험 화면은 일반 문제 화면과 다르게 보인다.
- 승급 시험에 통과하면 레벨이 올라간다.
- 앱을 껐다 켜도 현재 레벨과 레이트가 유지된다.

## 15. 다음 결정 사항

다음 단계에서 결정해야 할 내용:

- 일반 세션 문제 수를 3문제로 고정할지
- 승급 시험 문제 수를 5문제로 고정할지
- 레이트 기준을 80으로 시작할지
- 승급 시험 실패 시 레이트를 얼마나 낮출지
- 문제 유형을 선택형 위주로 시작할지
- 짧은 답변 평가에 AI를 언제 사용할지
