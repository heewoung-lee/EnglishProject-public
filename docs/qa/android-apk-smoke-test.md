# Android APK Smoke Test Checklist

## 목적

로컬 Android 빌드로 만든 APK를 실제 기기에서 빠르게 반복 검증한다. 이 체크리스트는 APK를 교체할 때마다 같은 기준으로 확인하기 위한 최소 수동 QA 기준이다.

## APK 설치

- APK 경로: `C:\Users\woung\Desktop\이력서\app-release.apk`
- portable 경로: `$env:USERPROFILE\Desktop\이력서\app-release.apk`
- 설치 명령:

```powershell
adb install -r "$env:USERPROFILE\Desktop\이력서\app-release.apk"
```

- APK SHA256 기록:

```powershell
Get-FileHash -Algorithm SHA256 "$env:USERPROFILE\Desktop\이력서\app-release.apk"
```

- 되돌릴 커밋 기록:

```powershell
git log --oneline -1
```

## 테스트 항목

### 처음 실행

- 앱을 완전히 종료한 뒤 다시 실행한다.
- 로컬 학습 상태가 로드되고, 현재 레벨에 맞는 문제가 바로 표시되는지 확인한다.
- 상단 레벨/영역, Rate, 설정 버튼이 서로 겹치지 않는지 확인한다.

### 영작 키보드

- 영작 문제에서 입력창을 누른다.
- 키보드가 올라와도 제출 버튼이 가려지지 않는지 확인한다.
- 긴 문장을 입력해도 입력창과 제출 버튼이 화면 밖으로 밀리지 않는지 확인한다.

### 회화 키보드

- 회화 문제에서 답변 입력창을 누른다.
- 채팅 목록이 최신 메시지로 이동하고, 입력창/전송 버튼이 키보드 위에 남는지 확인한다.
- 대화가 길어져도 마지막 메시지와 입력 영역이 서로 겹치지 않는지 확인한다.

### 채점 중 팝업

- 회화의 마지막 답변을 전송한다.
- AI 채점 결과가 나오기 전에 `채점 중` 팝업이 표시되는지 확인한다.
- 팝업이 떠 있는 동안 전송 버튼을 반복해서 누를 수 없는지 확인한다.

### 승급 시험

- Rate를 승급 기준 이상으로 만든다.
- 일반 문제 화면과 다른 승급 시험 화면으로 진입하는지 확인한다.
- 승급 시험 제출 버튼이 긴 문항에서도 하단에 유지되는지 확인한다.
- 결과 화면의 점수 표기가 `점수 X / Y`처럼 자연스러운지 확인한다.

### 설정 메뉴

- 오른쪽 상단 톱니바퀴 버튼을 누른다.
- `레벨초기화`와 `종료`가 표시되는지 확인한다.
- 레벨초기화 후 앱이 A1 상태로 다시 문제를 시작하는지 확인한다.
- 종료 버튼이 앱 종료 동작을 시도하거나, 지원되지 않는 환경에서는 안전하게 닫히는지 확인한다.

## PASS 기준

- 앱이 크래시 없이 실행된다.
- 처음 실행, 일반 문제, 영작 키보드, 회화 키보드, 채점 중 팝업, 승급 시험, 설정 메뉴 흐름이 모두 동작한다.
- 키보드가 올라온 상태에서 제출/전송 버튼이 가려지지 않는다.
- 상단 UI와 Rate/설정 버튼이 겹치지 않는다.
- 동일 APK SHA256과 되돌릴 커밋을 기록할 수 있다.

## FAIL 기준

- 앱 실행 직후 빨간 오류 화면이나 크래시가 발생한다.
- 영작 또는 회화에서 키보드가 제출/전송 버튼을 가린다.
- 회화 마지막 답변 뒤 채점 중 상태를 사용자에게 보여주지 않는다.
- 승급 시험에서 제출 버튼이 스크롤 안에 묻혀 조작하기 어렵다.
- 설정 메뉴가 열리지 않거나 레벨초기화가 동작하지 않는다.
- 테스트 증거를 남길 수 없을 정도로 APK 경로, APK SHA256, 되돌릴 커밋이 불명확하다.

## 증거 기록

- 스크린샷: 처음 실행, 영작 키보드, 회화 키보드, 채점 중 팝업, 승급 결과 화면
- APK SHA256: `Get-FileHash` 결과
- 되돌릴 커밋: `git log --oneline -1` 결과
- 기기 정보: Android 버전, 모델명, 키보드 종류
